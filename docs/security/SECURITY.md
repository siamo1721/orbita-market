## 3. Комментарии к двум строкам (по ТЗ)

### Строка 03 — пароль PostgreSQL в docker-compose

**Почему ТР:** пароль `postgres` захардкожен в `docker-compose.yml` и попадает в git. Gitleaks не сработал, потому что это не формат API-key/token, но для production это слабые учётные данные.

**Что делаем:** вынос переменых в .env и описываем .env.example

### Строка 04 — отсутствие аутентификации

**Почему риск (не FP):** любой, кто достучится до `:8080`, может указать произвольный `X-User-Id` и работать с чужим счётом/заказами.

**Почему не блокер для сдачи:** ТЗ п. 1.1 прямо говорит — «сервисы не реализуют полноценную аутентификацию; Gateway может просто проксировать X-User-Id».
